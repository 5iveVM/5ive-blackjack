# Security Policy

If you find a security issue, please do **not** open a public issue with exploit details.

## Reporting
- Send a private report with:
  - impacted component (`src/`, `client/`, `web/`, etc.)
  - reproduction steps
  - expected vs actual behavior
  - risk assessment (funds, account integrity, DoS, etc.)

## Scope Notes
- This project is educational and currently uses mock-chip game economics.
- Still, account validation and session authorization behavior are security-relevant and should be reported.

## Response Goals
- Acknowledge receipt quickly.
- Reproduce and triage.
- Patch and publish remediation notes when appropriate.
